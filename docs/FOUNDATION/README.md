# ENAF Foundation

The Foundation establishes the core concepts of the Enterprise Network Architecture Framework (ENAF).

It defines the philosophy, principles, lifecycle, governance, and practical guidance that form the basis for every other component of the framework.

Before using the lifecycle, templates, toolkits, or case studies, architects should become familiar with the concepts presented in this section.

---

## Purpose

The Foundation provides a common understanding of ENAF by answering the following questions:

- What is ENAF?
- Why does it exist?
- What principles guide architectural decisions?
- How does the framework operate?
- How does ENAF align with industry standards?
- How is architecture governed?
- What deliverables should architects produce?
- How should ENAF be applied in real projects?

These documents establish the common language and methodology used throughout the framework.

---

# Foundation Documents

| Document | Description |
|----------|-------------|
| **01 - Introduction** | Introduces ENAF, its vision, objectives, audience, and overall purpose. |
| **02 - What is ENAF** | Explains the framework, its scope, philosophy, and intended use. |
| **03 - Guiding Principles** | Defines the architectural principles that guide every decision throughout the lifecycle. |
| **04 - Architecture Lifecycle** | Describes the complete ENAF lifecycle from business strategy to continuous improvement. |
| **05 - Industry Alignment** | Demonstrates how ENAF aligns with established enterprise architecture, governance, security, cloud, and operational frameworks. |
| **06 - Governance Model** | Defines governance principles, responsibilities, decision-making, and architectural oversight. |
| **07 - Architecture Deliverables** | Defines the expected outputs produced during each stage of the ENAF lifecycle. |
| **08 - How to Use ENAF** | Provides practical guidance for applying ENAF to enterprise transformation initiatives. |
