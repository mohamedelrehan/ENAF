# Frameworks

## Purpose

Explain how established industry frameworks support the activities performed during the Strategy stage.

This document does not replace those frameworks. Instead, it identifies the guidance that architects should use while executing Strategy activities within ENAF.

---

# Framework Selection Principles

No single framework addresses every aspect of enterprise transformation.

ENAF combines guidance from multiple industry-recognized frameworks, using each where it provides the greatest value.

Examples include:

- TOGAF for enterprise architecture
- ArchiMate for architecture modeling
- COBIT for governance
- ITIL for service management alignment
- NIST CSF for cybersecurity
- SABSA for security architecture
- ISO 27001 for information security
- Microsoft CAF for cloud adoption
- AWS Well-Architected Framework
- Azure Well-Architected Framework

ENAF integrates these frameworks without requiring organizations to adopt all of them.

# Framework Contributions

## TOGAF

Supports business architecture, stakeholder management, architecture vision, and strategic alignment.

Use TOGAF primarily during:

- STR-01
- STR-02
- STR-03
- STR-04
- STR-05
- STR-09

---

## COBIT

Supports governance, investment decisions, value realization, and executive accountability.

Use COBIT during:

- STR-03
- STR-06
- STR-08
- STR-09

---

## ITIL

Supports alignment between strategy and future service operations.

Use ITIL when evaluating:

- Operational readiness
- Service ownership
- Support considerations

Primarily:

- STR-07

---

## NIST Cybersecurity Framework

Helps identify cybersecurity risks that should influence strategic planning.

Use during:

- STR-08

---

## SABSA

Provides guidance for integrating business-driven security requirements into enterprise strategy.

Use during:

- STR-02
- STR-08

---

## ISO 27001

Supports governance and compliance planning for information security.

Use during:

- STR-08

---

## Microsoft Cloud Adoption Framework

Useful when cloud transformation is part of the business strategy.

Supports:

- Cloud adoption planning
- Business justification
- Organizational readiness

Typically used during:

- STR-02
- STR-07

---

## AWS and Azure Well-Architected Frameworks

Provide strategic design principles for future cloud architectures.

Although detailed design occurs later, these frameworks help architects understand long-term operational considerations.

---

# Selecting Frameworks

Organizations are not expected to adopt every framework.

Architects should select the frameworks that best match:

- Industry
- Business maturity
- Regulatory requirements
- Technology landscape
- Organizational goals

ENAF remains framework-agnostic while promoting industry best practices.

---

# Framework Integration

ENAF acts as the orchestration layer.

Rather than replacing existing methodologies, ENAF coordinates guidance from multiple frameworks into a single enterprise architecture lifecycle.

---

## Summary

Industry frameworks provide valuable guidance for specific Strategy activities.

ENAF integrates these frameworks by identifying where each one adds value, enabling architects to apply proven practices without being constrained to a single methodology.
