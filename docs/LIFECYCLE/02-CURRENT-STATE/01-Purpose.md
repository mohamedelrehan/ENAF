# Purpose

The purpose of the Current State stage is to answer one fundamental question:

> **Where are we today?**

Following approval of the Strategy stage, Enterprise Architects begin building a complete understanding of the organization's existing business and technology environment.

Before designing the future, the current enterprise must be understood through evidence rather than assumptions.

The Current State stage establishes a trusted architectural baseline by assessing the organization's existing capabilities, technologies, operational practices, and governance across all Enterprise Assessment Domains.

This baseline provides the factual foundation required for every architectural decision that follows.

Without understanding the current environment, it is impossible to design an effective Target State, produce a realistic transformation roadmap, or deliver sustainable business outcomes.

---

# Why This Stage Matters

Enterprise environments evolve continuously.

Business expansion, mergers and acquisitions, cloud adoption, digital transformation, regulatory requirements, and technology modernization often create environments containing a mixture of modern platforms, legacy systems, duplicated capabilities, and inconsistent operational practices.

As a result, organizations frequently experience:

- Complex technology landscapes.
- Inconsistent standards.
- Technical debt.
- Operational inefficiencies.
- Security gaps.
- Limited visibility.
- Increasing operational costs.
- Business dependencies that are not fully understood.

The objective of the Current State stage is not to evaluate whether existing solutions are good or bad.

Instead, the objective is to understand how the enterprise currently operates and establish an accurate picture of the organization's existing capabilities before planning any future change.

---

# Enterprise Assessment Domains

ENAF introduces **Enterprise Assessment Domains** to ensure that the organization is assessed from a complete enterprise perspective rather than focusing on individual technologies.

Each domain contributes to the overall architectural baseline.

The domains assessed during this stage are:

- Business
- Applications
- Infrastructure
- Network
- Cloud
- Security
- Operations
- Governance

Together, these domains provide a comprehensive understanding of the enterprise and establish a common structure that is reused throughout the ENAF lifecycle.

---

# What This Stage Answers

The Current State stage provides clear answers to questions such as:

- How does the organization operate today?
- Which business capabilities currently exist?
- Which applications support critical business services?
- How are infrastructure and network services designed?
- Which cloud platforms are currently in use?
- How is security implemented?
- How are services operated and supported?
- Which governance processes currently exist?
- What risks and constraints affect the organization?
- Where does technical debt exist?
- Which capabilities should be preserved?
- Which capabilities require modernization?

These answers establish the baseline for every architectural decision that follows.

---

# Expected Outcome

By completing the Current State stage, the Enterprise Architecture team develops a comprehensive understanding of the existing enterprise.

The assessment identifies:

- Existing business capabilities.
- Technology assets.
- Enterprise dependencies.
- Operational maturity.
- Governance maturity.
- Strengths that should be retained.
- Weaknesses that require improvement.
- Risks that require mitigation.
- Technical debt that should be addressed during the transformation.

The result is a trusted Current State Assessment that provides a factual and validated baseline for defining the future Target State.

---

# Relationship to the ENAF Lifecycle

The Current State stage builds directly upon the Strategy stage.

Each stage of the ENAF lifecycle answers one fundamental question.

| Lifecycle Stage | Primary Question |
|-----------------|------------------|
| Strategy | **Why are we transforming?** |
| Current State | **Where are we today?** |
| Target State | **Where do we want to be?** |
| Architecture | **What should we build?** |
| Roadmap | **When and in what order should we deliver it?** |
| Implementation | **How do we build it?** |
| Operations | **How do we operate it?** |
| Governance | **How do we control it?** |
| Optimization | **How do we continuously improve it?** |

This progression creates a logical and traceable transformation journey, ensuring that every stage builds upon the outcomes of the previous one.

---

# Summary

The Current State stage establishes the architectural baseline for the enterprise transformation.

By answering the question **"Where are we today?"**, Enterprise Architects develop a complete understanding of the existing organization across all Enterprise Assessment Domains.

This understanding provides the evidence required to design the Target State, define the architecture, develop the transformation roadmap, and successfully deliver the enterprise transformation.

A successful transformation begins with understanding the enterprise as it exists today.
