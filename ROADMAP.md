# ENAF Roadmap

> **From Business Strategy to Operational Excellence**

## Purpose

This roadmap defines the planned development of the Enterprise Network Architecture Framework (ENAF).

ENAF will be developed incrementally. Each phase will produce practical, reusable, and reviewable architecture guidance before the next phase begins.

The roadmap may evolve as the framework matures, but changes should preserve the agreed repository structure and core methodology.

---

## Development Principles

ENAF development follows these principles:

- Business value before technical detail
- Architecture before implementation
- One document or capability at a time
- Review before approval
- Commit approved work as a stable baseline
- Avoid unnecessary restructuring
- Keep the methodology vendor-neutral
- Use practical enterprise examples
- Connect every phase to measurable business outcomes
- Continuously improve based on real-world use
